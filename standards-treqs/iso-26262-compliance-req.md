ISO-26262-6-R1: Software system requirements derived from system safety requirements
--

Software safety requirements shall be derived from technical safety requirements, and shall:

- Specify safety-related functional and non-functional behavior.
- Be verifiable through acceptance criteria.
- Specify system and hardware configurations.

ISO-26262-6-R2: Verification of software safety requirements
--

Each software safety requirement shall be verified using appropriate methods to demonstrate sustainability for software development, compliance with the system design and technical safety reqs, and consistency with the hardware-software interface.
